SafeSight: GPS-Denied Drone Navigation via LiDAR-Inertial Odometry

Robotics Software Engineer — Drone Autonomy · SafeSight

A modular perception device that provides real-time LiDAR-inertial odometry to PX4 as external vision — enabling position-hold and autonomous inspection in underground and industrial GPS-denied environments. Shipped across FAST-LIO2 and Point-LIO backends with tuned Fast DDS shared-memory transport.

Problem

Position control where GPS cannot reach

Underground tunnels, mines, and industrial interiors have no GPS and unreliable barometers. Without an external position source, PX4's EKF2 falls into inertial dead-reckoning within seconds — uncertainty explodes, position-hold drifts, and the drone becomes uncontrollable. SafeSight solves this by supplying continuous, high-rate LiDAR-inertial odometry as the drone's primary position-aiding source.

Architecture

LiDAR-inertial odometry → PX4 EKF2 pipeline

1 — Sensor synchronisation & calibration

LiDAR-to-IMU time synchronisation via hardware PPS and software timestamp alignment; extrinsic calibration between the Ouster OS1-128 and VectorNav VN-310; and a clean transform chain from sensor → body → PX4 local frame.

PPS sync → extrinsic cal → TF: sensor → body → PX4 local

2 — LiDAR-inertial odometry (multi-backend)

Shipped multiple backends — FAST-LIO2 and Point-LIO, tightly-coupled iterated-EKF estimators — each outputting 6-DOF pose at IMU rate with improved robustness in geometrically degenerate spaces.

OS1-128 + VN-310 → FAST-LIO2 / Point-LIO → 6-DOF pose

3 — High-throughput DDS transport

The OS1-128 at 20 Hz produces ~6–8 MB per scan. Configured Fast DDS with shared-memory (SHM) transport to move point clouds without packet loss — tuning maxMessageSize, fragment sizes, and kernel socket buffers for reliable intra-host delivery.

Fast DDS SHM → maxMessageSize tuning → kernel rmem_max

4 — PX4 external-vision integration

Streamed odometry to PX4 via the MAVLink ODOMETRY message as external-vision input to EKF2 — configuring EV aiding parameters, timestamp conventions, covariance scaling, and delay compensation, validated through estimator_aid_src_ev_* and vehicle_local_position.

LIO pose → MAVLink ODOMETRY → EKF2 EV fusion → position-hold

5 — Failsafe & robustness hardening

Diagnosed and resolved position-hold instability on odometry dropout: implemented failsafe gating so Position mode is blocked unless local position is valid, with automatic fallback to Altitude/Stabilised on aiding loss — validated with controlled no-odometry isolation flights.

EKF health monitoring → mode gating → graceful degradation
Diagnostic Deep-Dive

EKF2 failure analysis

Systematic flight-log analysis across baseline, failed, and controlled-isolation flights pinned down the position-hold failure signature — the moment aiding is lost and EKF2 slips into dead-reckoning:

IndicatorHealthyFailure
dead_reckoning0 (aiding active)1 (inertial drift)
xy_valid10 (position invalid)
ephLow, stableSaturating / diverging
xy_reset_counterStableIncrementing repeatedly

Root causes investigated: odometry stream dropout, timestamp/time-sync mismatches, covariance scaling, and insufficient update rate. Controlled isolation flights (odometry intentionally stopped) confirmed the signature matches "aiding missing" — closing the loop on the fix.

Work Area 1

Multi-sensor localization

GPS + IMU + Wheel Odometry Fusion
First-pass localization fusing RTK-GNSS, IMU, and wheel odometry for outdoor position estimation. Tuned fusion parameters and covariance weighting across different sensor vendors and terrain conditions.
RTK + IMU + wheel odom → fused pose → robot_localization / EKF
AMCL Tuning & Localization Methods
Extensive tuning of AMCL and other localization algorithms across different environments and sensor configurations. Developed generic approaches applicable to any UGV platform — not one-off configurations.
Map-based localisation → AMCL parameter tuning → cross-platform validation
Work Area 2

Navigation — Nav2 + custom plugins

GPS Waypoint Following
Implemented GPS waypoint following using Nav2 with a custom plugin for outdoor mission execution. Converts GPS coordinates to local frame and feeds Nav2's waypoint follower for autonomous route traversal.
GPS waypoints → local frame conversion → Nav2 waypoint follower
Generic Navigation Plugins
Building reusable Nav2 plugins for any UGV platform: generic controller tuning, costmap layer configuration, and recovery behaviours. Future work includes mesh-based navigation plugin and ConvNav plugin integration for unstructured terrain.
Generic Nav2 plugins → mesh/ConvNav (planned) → any UGV
Work Area 3

Mapping & environment reconstruction

Working on outdoor mapping pipelines using multi-LiDAR (Ouster, Livox, Hesai) data to generate maps for navigation and inspection. Addressing challenges of large-scale outdoor mapping: GPS-aided loop closure, point cloud density management, and map format compatibility with Nav2 costmap layers.

Work Area 4

Swarm systems & drone-UGV integration

Implementing multi-robot swarm coordination with DDS tuning for inter-robot communication. Working on integrating a quadruped (dog robot) with a drone for heterogeneous multi-robot operations. DDS discovery, namespace isolation, and bandwidth management across multiple robots on the same network.

Work Area 5

Infrastructure & tooling

Streaming Agent
Built a software agent that extracts telemetry data from robots and drones in real-time and streams it to Wakeb's cloud platform for monitoring, logging, and fleet management.
Simulation & DevOps
Extensive work in NVIDIA Isaac Sim and Gazebo (classic + Ignition) for testing navigation and localization before field deployment. Dockerised the full Clearpath software stack for reproducible deployments across the robot fleet.
Isaac Sim + Gazebo → Docker images → fleet deployment
DDS Middleware Tuning
Production Fast DDS configuration across all platforms: discovery tuning for multi-robot networks, transport configuration for high-bandwidth LiDAR data, and QoS policy management for reliable sensor data delivery.
Fast DDS profiles → discovery → QoS → multi-robot network

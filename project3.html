Vision-Based Seam Tracking for Robotic Welding

Real-time closed-loop perception system that detects and tracks welding seams while dynamically correcting robot trajectory during operation — maintaining precise weld placement despite part tolerances and positioning errors in harsh industrial environments.

Real-Time Vision Closed-Loop Control Coordinate Transforms Camera-Robot Calibration Seam Detection Low-Latency Pipeline Industrial Robotics
Problem

Precision welding under real-world uncertainty

Pre-programmed robot paths assume perfect part placement, but in practice parts have tolerances, fixturing errors, and thermal distortion that shift the actual seam position. Without real-time correction, weld quality degrades — causing rework, scrap, or structural failures. The system must operate under glare, arc light, and reflections that defeat conventional vision approaches.

Architecture

Closed-loop perception → correction pipeline

1 — Vision Sensor Integration
Vision sensor mounted near the welding tool (TCP) for direct seam observation. Optics and sensor selected for required accuracy and working distance, hardened against arc light, glare, and spatter.
Sensor selection → mounting near TCP → environmental hardening
2 — Seam Detection & Feature Extraction
Detect seam position using structured visual features. Robust under challenging industrial conditions — specular reflections, varying surface finish, and welding arc interference are handled at the image processing level.
Image acquisition → feature extraction → seam localisation
3 — Camera-to-Robot Calibration
Full calibration chain: camera frame ↔ TCP ↔ robot base. Seam position in image coordinates is transformed into the robot's coordinate frame for direct path correction.
Hand-eye calibration → T_cam→TCP → T_TCP→base
4 — Real-Time Tracking & Online Correction
Continuous seam position estimation during robot motion. Low-latency communication feeds correction offsets to the robot controller, which adjusts the planned path on-the-fly to maintain alignment throughout the entire weld.
Perception loop → offset computation → controller correction @ cycle rate

Project 01 • 3D Reconstruction • Dental Geometry Processing

Automated Dental Shell Generation Pipeline

End-to-end Python pipeline that converts raw intraoral scans into watertight, multi-thickness shells (soft / medium / hard) ready for CAD export and 3D printing — fully automated, no manual mesh editing required.

Intraoral Scans Surface Reconstruction Signed Distance Fields Python Open3D libigl PyMeshFix Poisson Reconstruction DBSCAN Ray Casting
Problem

From noisy scans to printable shells

Raw intraoral scans contain flat scanner artefacts, self-occluding geometry, and non-manifold edges. Clinicians need watertight shells at precise thicknesses (0.4 mm–1.6 mm) for aligner and retainer fabrication — a process that previously required hours of manual mesh editing per case.

Architecture

5-stage processing pipeline

1 — Scan Clean-up enhance_input.py
Remove triangles whose normals are within ±1.9° of any principal axis (flat scanner artefacts). Discard clusters < 5,000 triangles. Run PyMeshFix with joincomp=True for watertight repair.
Axis-aligned face removal → clustering → MeshFix
2 — Offset Surface Generation offset.py
Generate iso-surfaces at clinically defined distances using libigl's marching-tetrahedra on a 1000³ grid. A signed-distance filter retains only faces with positive SDF, guaranteeing a purely outer shell.
libigl offset_surface → SDF filter → inner/outer STLs
3 — Centre-Band Detection & Trimming center_spheres_raycasting.py
Place a virtual origin inside the arch, fire 360° planar rays every 13°, then seed marker spheres at forward/backward hit gaps. From each sphere, 500k isotropic rays mark and remove occluded triangles. DBSCAN extracts the largest clean cluster.
360° ray-casting → sphere seeding → DBSCAN clustering
4 — Ridge Bending (Experimental) creating_ridges.py
Detect multi-band rays (≥5 surface hits), locally displace vertices downward up to 0.5 mm to visualise occlusal ridges. Retained for future morphology studies.
Multi-band ray detection → local vertex displacement
5 — Watertight Reconstruction poisson.py
Estimate normals (radius 0.3 mm, max 30 neighbours), flip inner normals outward, merge point clouds, and run Poisson reconstruction at depth 10. Low-density triangles (≤ 0.02) are discarded to remove skirts. Output: final watertight STL.
Normal estimation → flip + merge → Poisson depth 10 → density filter

Challenges & Trade-offs

Known limitations

Raycasting outliers: The 360° ray-casting stage can produce false-positive marked triangles on thin enamel (visible as detached clusters). While DBSCAN filters the largest cluster, edge cases in paediatric scans with narrow anatomy can over-trim. Mitigation: tunable gap_threshold_mm and cluster_min_size.

Offset self-intersections: On tight curvature regions, marching-tet offset can self-intersect. A voxel dilation/erosion fallback is planned.
